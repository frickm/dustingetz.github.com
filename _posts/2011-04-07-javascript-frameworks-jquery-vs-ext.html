Javascript frameworks: JQuery vs EXT 

jQuery is for working with web pages. Ext JS is for writing GUI apps. That's a powerful difference.  

 

From their respective websites: 

"jQuery is a fast and concise JavaScript Library that simplifies HTML document traversing, event handling, animating, and Ajax interactions for rapid web development." [jquery.com] 

"Ext JS is the developer's choice for building powerful desktop web applications using JavaScript and web standards." [sencha.com] 

 

If you're writing a blog application, where you want to publish traditional HTML content to various formats (browsers, RSS, mobile), be searchable from google and stuff like that, jQuery is a good choice. It lets you add special effects like tooltips, transitions, selective highlights, form logic, basically just enhancements to your existing HTML. Its also very good at manipulating existing HTML ("traversing the DOM" [1]), which makes it ideal for writing plugins for existing web pages. I wrote an experimental bookmarklet[2] which analyses the hackernews homepage to identify and highlight posts with high commenting activity. jQuery was great for extracting comment and vote counts[3] from the hackernews HTML. Check out the jQuery examples page[4]. 

 

Ext JS is suited for web applications, like a calendar. Users will interact with your application like a desktop GUI app, and instead of thinking in HTML, you think in terms of GUI components, click events and data sources. Ext JS will generate all the underlying HTML for you. Your application won't feel like a web page, it will feel more like a desktop program. Of course Ext apps are hosted in a browser, but things like opening links in another window, and the forward/back buttons, often won't make sense. You can see this by googling for "extjs api", clicking through to the Ext API docs, browsing around then clicking 'back' on your browser: it won't take you to the previous doc you were looking at, but will take you back to google. Clicking 'forward' will bring you back to the Ext docs, but you don't go back to all your open pages. Joel calls this a leaky abstraction[5]. you can work around this[6], but it's against the grain of the framework. Check out the Ext JS examples page[7].
